# Cursor CLI Integration Notes

## Source And Fork

- Upstream: Cursor (Anysphere) — proprietary closed-source binary distributed
  via `https://cursor.com/install`.
- **No fork**: source is not available. All ALE adaptation lives in
  `deployer.py`.
- **No OpenRouter routing**: `cursor-agent` validates keys against
  Cursor's own backend. Setting `OPENAI_API_BASE`, `ANTHROPIC_BASE_URL`, etc.
  is silently ignored.

## Install

```bash
# Linux — installs to ~/.local/bin/cursor-agent
curl -fsSL https://cursor.com/install | bash
```

Expected binary paths:
- `~/.local/bin/cursor-agent` → `~/.local/share/cursor-agent/versions/<v>/cursor-agent`

The CLI is a wrapper that invokes bundled node + JS bundle. Version
command: `cursor-agent --version`.

The deployer auto-installs if the binary is not on PATH.

## CLI Invocation

```bash
cursor-agent -p \
  --model claude-4.6-sonnet-medium \
  --output-format stream-json \
  --force --approve-mcps --trust --sandbox disabled
```

Required flags:
- `-p` (`--print`): non-interactive headless mode (prompt from stdin)
- `--output-format stream-json`: NDJSON event stream
- `--model <id>`: Cursor's own model id
- `--force --approve-mcps --trust --sandbox disabled`: full headless permission bypass

## Authentication

`cursor-agent` authenticates via `~/.config/cursor/auth.json`, NOT
`CURSOR_API_KEY`. The auth.json contains:

```json
{
  "accessToken":  "<JWT>",
  "refreshToken": "<token>",
  "apiKey":       "<crsr_...>"
}
```

Generated by `cursor-agent login` (OAuth browser flow).

### Auth passthrough into sandbox

The deployer's `_setup_cursor_auth()` writes auth.json inside the
sandbox from env vars:
1. `CURSOR_AUTH_JSON` — inline JSON content (preferred)
2. `CURSOR_AUTH_JSON_PATH` — file path (lifecycle reads and materializes
   content into `CURSOR_AUTH_JSON`)

### macOS Keychain extraction

On macOS, tokens are in Keychain, not a file. Extract with:
```bash
security find-generic-password -s "cursor-access-token"  -a "cursor-user" -w
security find-generic-password -s "cursor-refresh-token" -a "cursor-user" -w
security find-generic-password -s "Cursor Safe Storage"  -a "Cursor Key"  -w
```

### Token refresh

OAuth `accessToken` JWTs expire (typically minutes). `cursor-agent`
auto-refreshes via the `refreshToken` on startup. Pushing a recent
auth.json is enough unless the `refreshToken` itself expired (days/weeks).

## Output Format

NDJSON events, one JSON object per line:

```jsonl
{"type":"system","subtype":"init","apiKeySource":"env","cwd":"...","session_id":"<uuid>","model":"Sonnet 4.6 1M"}
{"type":"user","message":{"role":"user","content":[{"type":"text","text":"<prompt>"}]}}
{"type":"assistant","message":{"role":"assistant","content":[{"type":"text","text":"<reply>"}]}}
{"type":"tool_call","subtype":"started","call_id":"toolu_…","tool_call":{"mcpToolCall":{"args":{...}}}}
{"type":"tool_call","subtype":"completed","call_id":"toolu_…","tool_call":{"mcpToolCall":{"result":{...}}}}
{"type":"result","subtype":"success","duration_ms":N,"usage":{"inputTokens":N,"outputTokens":N,...}}
```

Key shape differences from Claude Code stream-json:
- `tool_call` is its own top-level event (not a `tool_use` content block)
- Tool payload nested at `event.tool_call.<kind>ToolCall.{args,result}`
- Usage fields are camelCase (`inputTokens` etc.)
- Cursor echoes user prompt as a `{"type":"user"}` event (parser suppresses echo)

## Tool Classification

### Permission scope grammar

`cli-config.json` accepts five scope tokens:

| Token | Matches | Example |
|---|---|---|
| `Shell(<pattern>)` | shell commands | `Shell(*)` |
| `Read(<glob>)` | file reads | `Read(**)` |
| `Write(<glob>)` | file writes | `Write(**)` |
| `WebFetch(<domain>)` | web fetch | `WebFetch(*)` |
| `Mcp(<server>, <tool>)` | MCP tools | `Mcp(*:*)` |

### Built-in tools (16)

`Shell`, `Read`, `Write`, `Glob`, `Grep`, `StrReplace`, `Delete`,
`EditNotebook`, `TodoWrite`, `WebSearch`, `WebFetch`, `GenerateImage`,
`Task`, `ListMcpResources`, `FetchMcpResource`, `SwitchMode`.

### CUA MCP tools (14)

Namespaced as `cua-<tool>` in the stream: `cua-screenshot`, `cua-click`,
`cua-type`, `cua-key`, `cua-key_down`, `cua-key_up`, `cua-hold_key`,
`cua-mouse_move`, `cua-mouse_down`, `cua-mouse_up`, `cua-drag`,
`cua-scroll`, `cua-wait`, `cua-cursor_position`.

## Known Issues

### BYOK not possible

Cursor CLI rejects non-Cursor keys. OpenRouter routing is not supported.
Model selection is limited to Cursor's catalog.

### approvalMode must be "unrestricted"

With `"allowlist"`, cursor-agent may silently fail to load MCP servers.
The deployer uses `"unrestricted"` (canonical YOLO mode).

### No `--max-turns` flag

Runtime cap is the deployer-side `timeout_s`. The `max_turns` config
field is informational only.

### No `--mcp-config` flag

MCP servers are auto-discovered from `~/.cursor/mcp.json` only.
