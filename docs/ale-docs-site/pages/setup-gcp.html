Set up Google Cloud

One-time provisioning to run ALE at benchmark scale on Google Cloud: a project, a service account, the two sandbox images, networking, and a results bucket. About 20 minutes, mostly waiting on the image copies. The next page wires up an experiment.

Free trial runs Linux only
The full benchmark spans Linux and Windows tasks, and a Google Cloud free trial cannot create Windows VMs. To run the whole benchmark, activate a full billing account (Console → Billing → Activate full account; your $300 credit still applies). The Linux-only subset runs fine on the trial.

Before you start

  • A Google Cloud account.
  • The gcloud CLI installed, then signed in: gcloud auth login.

Clone the repository and install. --all-packages is required: it pulls in each agent's dependencies and the task-side evaluation package (tasks/pyproject.toml) used to grade locally — plain uv sync installs neither. 

git clone git@github.com:rdi-berkeley/agents-last-exam.git
cd agents-last-exam
uv sync --all-packages --extra dev

Provision the project

Edit the two variables at the top, then paste the block. It creates the project, links billing, enables APIs, makes a service account with a key, copies both images, sets up the network, and creates the results bucket. 

# ─── edit these ────────────────────────────────────────────────
export GCP_PROJECT="ale-$(whoami)-$(date +%s | tail -c 5)"   # unique, lowercase, ≤30 chars
export GCP_REGION="us-central1"                              # keep within the env config's zones
# ───────────────────────────────────────────────────────────────
export GCP_SA_NAME="ale-runner"
export GCP_SA_EMAIL="${GCP_SA_NAME}@${GCP_PROJECT}.iam.gserviceaccount.com"
export GCP_BUCKET="${GCP_PROJECT}-ale-results"

cd /agents-last-exam          # from the repo root

# 1. project + billing + APIs
gcloud projects create "$GCP_PROJECT" --name="ALE"
gcloud config set project "$GCP_PROJECT"
gcloud billing accounts list
read -p "billing-account ID from the list above: " BILLING_ID
gcloud billing projects link "$GCP_PROJECT" --billing-account="$BILLING_ID"
gcloud services enable compute.googleapis.com storage.googleapis.com

# 2. service account + roles + JSON key
gcloud iam service-accounts create "$GCP_SA_NAME" --display-name="ALE runner"
for role in compute.instanceAdmin.v1 compute.networkUser iam.serviceAccountUser \
            storage.objectViewer serviceusage.serviceUsageConsumer ; do
  gcloud projects add-iam-policy-binding "$GCP_PROJECT" \
    --member="serviceAccount:$GCP_SA_EMAIL" --role="roles/$role" --condition=None
done
mkdir -p secret
gcloud iam service-accounts keys create secret/gcp_key.json --iam-account="$GCP_SA_EMAIL"

# 3. copy BOTH published images (Linux + Windows) into your project (~3 min each)
for img in ale-ubuntu22 ale-win10 ; do
  gcloud compute images create "$img" \
    --source-image="$img" --source-image-project=agenthle-488519
done

# 4. network + firewall (cua-server is on tcp:5000; RDP is optional, for Windows debug)
gcloud compute networks create ale-vpc --subnet-mode=auto
gcloud compute firewall-rules create ale-allow-cua --network=ale-vpc \
  --direction=INGRESS --allow=tcp:5000 --source-ranges=0.0.0.0/0
gcloud compute firewall-rules create ale-allow-rdp --network=ale-vpc \
  --direction=INGRESS --allow=tcp:3389 --source-ranges=0.0.0.0/0

# 5. results bucket for batch output, with write access for the runner
gcloud storage buckets create "gs://$GCP_BUCKET" \
  --project="$GCP_PROJECT" --location="$GCP_REGION" --uniform-bucket-level-access
gcloud storage buckets add-iam-policy-binding "gs://$GCP_BUCKET" \
  --member="serviceAccount:$GCP_SA_EMAIL" --role="roles/storage.objectAdmin"

Every command is idempotent except gcloud projects create; if a step fails, fix it and re-run just that step. Keep the bucket name gs://<GCP_PROJECT>-ale-results handy — you'll point the environment config at it on the next page.

Notes

  • Region / zones. The gcloud snapshots in configs/environments/environment.yaml list fixed fallback zones (us-central1 / us-east1, plus us-west1 for Linux). Keep GCP_REGION within those, or edit the zones: lists to match.
  • Cost cap. A full-benchmark run of a frontier agent costs single-digit dollars per task and hours of wall-clock. Set a budget alert under Console → Billing → Budgets & alerts — alert-only (GCP won't auto-stop), but it catches a runaway sweep.
  • Licensed tasks. A handful of tasks need commercial software pre-installed on a custom image. Until that image is published, run the unlicensed track, which uses the images above as-is.
