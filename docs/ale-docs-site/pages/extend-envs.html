Add an environment

A new backend — another cloud, or your own hardware — is a Provider: it boots a sandbox and hands the framework a live cua-server endpoint. Today gcloud and static ship; more are on the way.

Roadmap

  • Local — VMware & QEMU. Boot the sandbox images on your own machine to run and debug tasks without the cloud.
  • More clouds — Azure, AWS, Aliyun. The same ephemeral-VM lifecycle on other providers, for capacity and region choice.
Want one sooner?
Implement the small Provider contract (acquire / release / open_session) and copy a reference — providers/gcloud.py (boot a VM) or providers/static.py (wrap an existing endpoint, e.g. a VM you brought up by hand). The contract is on Sandbox & provider.
